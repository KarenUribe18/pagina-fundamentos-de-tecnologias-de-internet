Puertos de red.

Puertos de red TCP.

El protocolo TCP permite establecer la conexión e intercambio de datos en la red definiendo el proceso integro desde que se envían los datos hasta que son recibidos precisando una conexión y una autorización entre el emisor y el receptor antes de iniciarse la transferencia de datos.

El protocolo TCP es como el correo postal. Cuando enviamos información importante, como un correo electrónico, donde no podemos permitirnos una pérdida de datos, usamos TCP. Es muy confiable porque asegura que todos los datos lleguen en el orden correcto y sin errores. Como si cada paquete de datos tuviera un recibo de entrega.

El protocolo TCP es un protocolo conectivo, fiable y orientado a conexión, esto significa que es capaz de retransmitir los segmentos de paquetes en caso de que haya alguna pérdida desde el origen al destino. El protocolo TCP para establecer la conexión realiza el 3-way handshake, con el objetivo de que la conexión sea lo más fiable posible. Si estamos utilizando algún protocolo en la capa de aplicación como HTTP, FTP o SSH que todos ellos utilizan el protocolo TCP, en la primera comunicación se realizará este intercambio de mensajes.

Un aspecto muy importante de los puertos TCP, es que existe un rango de puertos desde el 49152 al 65535 que son los puertos efímeros, es decir, con cada conexión de origen que nosotros realicemos, se utilizan estos puertos que son dinámicos. Por ejemplo, si realizamos una petición a una web, el puerto de origen estará en este rango 49152-65535, y el puerto de destino será el 80 (HTTP) o 443 (HTTPS).

Si utilizas el protocolo FTP, hoy en día se utiliza siempre FTP PASV, por lo que no solamente es necesario abrir el puerto TCP 21 para control, sino que deberemos abrir un rango de puertos para el intercambio de archivos entre el cliente FTP y el servidor FTP, de lo contrario, no podremos empezar a realizar las transferencias de datos.

Y estos serían los puertos más usados e importantes cuando hacen uso del protocolo TCP. En los equipos siempre los tendremos abiertos a no ser que un firewall lo esté cerrando explícitamente, pero en el router deberemos abrir todos estos puertos (port-forwarding o también conocido como reenvío de puertos) ya que estamos en un entorno NAT, y todos los puertos están cerrados.

Puertos de red UDP.

El protocolo UDP cuyas siglas vienen de User Datagram Protocol es una alternativa más simple y rápida si la comparamos con TCP que permite la transmisión si conexión de datagramas en redes basadas en protocolo IP.

El protocolo UDP es más rápido pero menos confiable. Se utiliza para actividades como transmisiones de vídeo en directo o para juegos online, donde la pérdida de algunos datos no supone un gran problema. Como una conversación telefónica, donde puedes perder algunas palabras, pero esto no supone un problema en la recepción del mensaje.

El protocolo UDP funciona sobre la capa de transporte y está diseñado para trasmitir datagramas de forma rápida en redes IP. Principalmente de utilizad para conexiones VPN, Consultas DNS, para conexión VOIP y también para streaming de audio y de video.

La ausencia de controles de transmisión hace que el protocolo UDP no ofrezca seguridad ni integridad de los datos, por ello son los propios servicios UDP los que deben aplicar medidas de protección y corrección.

Puertos del protocolo HTTP y HTTPS.

Los puertos HTTPS son puertos de red dedicados que permiten a los usuarios de Internet transmitir datos a través de una conexión segura cifrada utilizando un certificado SSL/TLS.

Más utilizados HTTP y HTTPS:

Puerto 80 - S utiliza de manera predeterminada para la comunicación entre los navegadores web (clientes) y los servidores web. Este puerto es el estándar para las solicitudes y respuestas HTTP en la World Wide Web.

Puerto 443 - El Grupo de Trabajo de Ingeniería de Internet (IETF) reconoce el número de puerto TCP 443 como el protocolo HTTPS predeterminado. Este proporciona un algoritmo de cifrado para intercambiar información entre servidores web y navegadores. El puerto 443 funciona asegurando los paquetes de tráfico de red antes de que ocurra la transmisión de datos.

Como puerto dedicado para la navegación web, este utiliza un certificado SSL/TLS para cifrar el texto original en un algoritmo y luego lo convierte en texto cifrado antes de enviarlo al servidor. Esto evita el espionaje no deseado y la intercepción de datos transmitidos a través del protocolo.

Puerto 8443 - El número de puerto 8443 es un puerto HTTPS alternativo y un protocolo principal que utiliza el servidor web Apache Tomcat para abrir el servicio de texto SSL.

Además, este puerto se utiliza principalmente como protocolo de conexión de autenticación de cliente HTTPS.

El puerto proporciona tráfico cifrado al generar un par de claves de autenticación para el usuario que se mantiene dentro del navegador web. Luego, el servidor web verificará la autenticidad de la clave privada antes de establecer una conexión segura.

Puertos del protocolo SMTP.

El protocolo simple de transferencia de correo o SMTP (Simple Mail Transfer Protocol) es un protocolo utilizado por servidores SMTP para enviar emails a través de internet.

Un puerto SMTP es un puerto de red que actúa como punto de entrada para conexiones SMTP. Es decir, es como una puerta a la que llaman los servidores SMTP para entrar y distribuir los correos electrónicos a los receptores. En otras palabras, es como la puerta de un piso al que llama el cartero para entregar la correspondencia.

En la actualidad,los puertos SMTP más habituales son:

Puerto 25. Puerto de protocolo simple de transferencia de email: el puerto 25 está bloqueado en muchas redes por el spam que históricamente se ha retransmitido desde ordenadores y servidores que han sido objeto de un uso fraudulento. Por eso, muchos ISP y proveedores de servicios bloquean o restringen las conexiones SMTP en el puerto 25.

Puerto 587. Puerto de envío de mensajes de email: en la actualidad, el puerto 587 se usa para enviar los emails de forma segura y garantizar que llegan a su destino. La mayoría de los softwares cliente están configurados para usar este puerto para enviar tus mensajes.

Casi todos los servidores de email admiten este puerto, pero aunque el servidor de correo lo admita, puede que esté o no abierto para la transmisión de correo electrónico.

Puerto 465. Puerto SMTP autenticado con SSL: la IANA asignó el puerto 465 en un principio para hacer una versión cifrada del SMTP llamada SMTPS. A finales de 1998, la IANA reasignó este número de puerto a un servicio nuevo. Aún así, muchos servicios siguen ofreciendo la interfaz SMTPS en el puerto 465.

El objetivo del puerto 465 es establecer un puerto para que el protocolo SMTP opere con la capa de puertos seguros (SSL). El protocolo SSL suele usarse para cifrar comunicaciones en internet. Por lo general, usarás este puerto solo si tu aplicación lo exige.

Puerto 80. Puerto de Protocolo de transferencia de hipertexto: el puerto 80 es el número de puerto asignado al protocolo de transferencia de hipertexto (HTTP), el protocolo de comunicación más usado en internet. Es el puerto desde el que un ordenador envía y recibe comunicaciones y mensajes basados en cliente web desde un servidor web, y se usa para enviar y recibir páginas o datos HTML. Este es el puerto al que el servidor “escucha” o espera recibir de un cliente web.

Puerto 443. Puerto de Protocolo seguro de transferencia de hipertexto (HTTPS): el puerto 443 funciona de forma análoga al puerto 80, sólo que asignado a HTTPS (la versión segura de HTTP). Presenta las mismas ventajas y desventajas que el puerto 80, pero con una capa de seguridad añadida.

Puerto 588. Puerto alternativo de envío de correos electrónicos: De forma similar al puerto 587, el puerto 588 se usa para enviar los emails de forma segura y garantizar que llegan a su destino. Con las mismas características que el puerto 587, el puerto 588 se utiliza como alternativa cuando el puerto 587 se encuentra cerrado.

Puerto 2525. El otro puerto alternativo: El puerto 2525 no es un puerto SMTP oficial, y ni el IEFT ni la IANA lo sancionan. Casi todos los ESP admiten el uso del puerto 2525, aunque no sea un puerto SMTP oficial. Puede usarse como alternativa a los puertos 587 y 588 para SMTP en el caso de que los demás estén bloqueados.

El puerto 2525 es posiblemente el más usado por los usuarios que se alojan en Google Computer Engine y experimentan problemas de conectividad con el puerto 587 con el envío SMTP.

Puertos del protocolo POP3.

La abreviatura POP3 significa Post Office Protocol versión 3, que proporciona acceso a una bandeja de entrada almacenada en un servidor de email. Ejecuta las operaciones de descarga y eliminación de mensajes. Por lo tanto, cuando un cliente POP3 se conecta al servidor de correo, recupera todos los mensajes del buzón. Luego los almacena en tu ordenador local y los elimina del servidor remoto.

Gracias a este protocolo, también puedes acceder a los mensajes localmente en modo fuera de línea. Los clientes POP3 modernos te permiten mantener una copia de tus mensajes en el servidor si seleccionas explícitamente esta opción.

Los puertos POP3 son:

Puerto 110. Es el puerto POP3 predeterminado y no está cifrado.

Puerto 995. Para conexiones seguras y el que recomendamos emplear siempre que sea posible.

    Puertos del protocolo MIME.

    MIME se utiliza principalmente en el contexto de correo electrónico para permitir la inclusión de contenido multimedia, como imágenes, audio y video, en los mensajes de correo electrónico. Los servidores de correo electrónico utilizan protocolos como SMTP (Simple Mail Transfer Protocol) para enviar correos electrónicos y POP3 (Post Office Protocol 3) o IMAP (Internet Message Access Protocol) para recuperarlos, pero MIME se encarga de la forma en que se codifica y presenta el contenido dentro de esos mensajes.

    El protocolo MIME (Multipurpose Internet Mail Extensions) no utiliza puertos específicos, ya que es un estándar utilizado para la codificación y el intercambio de mensajes de correo electrónico, así como para otros tipos de datos multimedia a través de Internet.

    En resumen, no hay puertos específicos para el protocolo MIME en sí,los puertos que se utilizan en el contexto del correo electrónico suelen ser los puertos estándar asociados con los protocolos mencionados (SMTP, POP3, IMAP), que son el puerto 25 para SMTP, el puerto 110 para POP3 y el puerto 143 para IMAP, entre otros.

    Puertos del protocolo FTP.

    FTP funciona en dos canales para intercambiar información: uno para comandos y otro para datos. Esto requiere dos puertos para que FTP funcione: el canal de comandos y el canal de datos.

    El canal de comandos funciona en el puerto 21, que acepta conexiones de clientes y gestiona el paso de comandos. El canal de comandos permanece abierto durante toda la sesión FTP hasta que el cliente envía QUIT o el servidor se desconecta forzosamente por inactividad o cualquier otro motivo posible.

    El canal de datos utiliza un puerto temporal a la escucha del servidor (modo pasivo) o del cliente (modo activo). Este canal es responsable de compartir datos reales entre el servidor y el cliente en forma de listados de directorios y transferencias de archivos.

    A diferencia del canal de comandos, el canal de datos permanece abierto sólo mientras dura la transferencia de archivos, y en cuanto ésta finaliza, el canal de datos se cierra. Para la transferencia simultánea de varios archivos o listados de directorios, se necesitan varios puertos de canal de datos.

    FTP es un protocolo intrínsecamente inseguro, ya que tanto los canales de comandos como los de datos transfieren información de forma no cifrada, lo que es fácilmente interceptable por cualquiera que utilice ataques man-in-the-middle.

    Tiene dos tipos de puertos:

    Puerto 21. Conexión de control: se utiliza para establecer la conexión de control entre el cliente FTP y el servidor FTP. La comunicación en el puerto 21 se utiliza para enviar comandos y recibir respuestas, como la autenticación del usuario, la navegación por directorios y la gestión de transferencias de archivos.

    Puerto 20. Conexión de datos: se utiliza para la transferencia de datos en sí. Cuando se realiza una transferencia de archivos o directorios, el servidor FTP abrirá una conexión de datos saliente desde su puerto 20 al puerto efímero del cliente (un puerto temporal asignado por el cliente). Esto permite la transferencia efectiva de archivos entre el cliente y el servidor.

    Puertos del protocolo FTPS.

    Las siglas FTPS provienen de FTP over SSL y no es más que el protocolo estándar FTP transmitido sobre una conexión segura. Inicialmente la conexión segura hacía referencia únicamente SSL (Secure Sockets Layer), pero con el paso del tiempo y la aparición del protocolo TLS (Transport Layer Security), también soporta este último.

    Un aspecto que puede resultar negativo de este protocolo es que por el hecho de utilizar más de un canal puede resultar problemático con algunas configuraciones en los firewalls, así que es posible que si tenemos un firewall en funcionamiento e implementamos un servidor FTPS nos veremos obligados a revisar su configuración.

    Este protocolo hace uso de dos canales diferentes: uno de control y otro de transferencia, y tiene dos variantes bien diferenciadas:

    FTP implícito sobre SSL.Recibe este nombre porque la conexión cifrada es implicita, es decir, ocurre desde un primer instante, sin necesidad de solicitarla, motivo por el cual además está desaconsejada (deprecated).

    Puertos del protocolo SFTP.

    SFTP (SSH File Transfer Protocol o Protocolo de Transferencia de Archivos SSH) es un protocolo de transferencia de archivos construido sobre la capa de transporte SSH (Secure Shell), que se utiliza para mover de forma segura grandes cantidades de información a través de una conexión a Internet.

    SFTP utiliza SSH para establecer una conexión segura y autenticada, y proporcionar a las organizaciones un mayor nivel de protección durante la transferencia de archivos . Utiliza la autenticación SSH y las funcionalidades de encriptación para proteger los archivos durante el proceso de transferencia.

    SFTP solo necesita un puerto para establecer una conexión con el servidor:

    Puerto 22. Conexión: el cliente se conecta al servidor a través del puerto 22, y todas las operaciones de transferencia de archivos y comandos de administración se realizan a través de esta única conexión segura.

    Puertos del protocolo BitTorrent.

    El BitTorrent es un protocolo de intercambio de datos de forma descentralizada. Es un sistema que siempre ha gozado de mucha popularidad para intercambiar archivos de todo tipo, y aunque la descarga directa le quitó protagonismo durante unos años, ahora vuelve a ser un protocolo muy utilizado. Hay páginas como las de distribuciones GNU/Linux y algunas instituciones que lo utilizan para permitirte descargar sus archivos.

    El protocolo BitTorrent pertenece a la familia de protocolos P2P, que significa Peer-to-Peer. Un protocolo P2P es aquel en el que los clientes se conectan directamente entre ellos sin pasar por un servidor central.

    Puertos comunes utilizados por el protocolo BitTorrent:

    Puerto 6881 - 6889: los clientes de BitTorrent utilizan un rango de puertos dentro de esta gama para establecer conexiones de transferencia de datos entre pares. Sin embargo, algunos proveedores de servicios de Internet (ISP) pueden limitar o priorizar el tráfico en estos puertos, lo que puede afectar el rendimiento de las descargas.

    Puerto 51413: muchos clientes de BitTorrent ahora utilizan el puerto 51413 como puerto predeterminado para conexiones de datos. Esto se debe a que algunos ISP pueden bloquear o limitar el tráfico en los puertos del rango 6881-6889, por lo que el puerto 51413 se ha vuelto más popular en los últimos años.

    Puertos del protocolo eDonkey.

    Es el nombre de una red de intercambio de archivos P2P, su nombre deriva del programa original creado para la misma. El nombre del cliente oficial es eDonkey2000, el cual en la actualidad ha dejado de funcionar. Dicho cliente tenía la capacidad de conectarse tanto a la red eDonkey como a Overnet.

    El sistema de intercambio de archivos eDonkey se puede clasificar como una red semicentralizada. Si bien existen servidores que manejan la información que transita por dicha red, no existe ninguno que sea principal. De esta forma, si un servidor deja de funcionar, otro puede emerger de la nada para suplantarlo. Incluso los usuarios puede levantar sus propios servidores.

    Los puertos más comunes asociados con el protocolo eDonkey:

    Puerto 4661 (TCP): los clientes se conectan a los servidores en este puerto para buscar y obtener información sobre archivos compartidos por otros usuarios.

    Puerto 4662 (TCP): es el puerto principal para las transferencias de datos entre pares (peer-to-peer). Los clientes se conectan entre sí a través de este puerto para descargar y cargar archivos.

    Puerto 4672 (UDP): se utiliza para funciones auxiliares, como la búsqueda de servidores eDonkey y la comunicación entre servidores. También se utiliza para realizar solicitudes de información sobre archivos a servidores de búsqueda.

    Puertos del protocolo FastTrack.

    Proporciona instrucciones remotas para la la incorporación de núcleos, lo que implica el aprovisionamiento de servicios, la configuración de inquilinos y la integración de identidades.

    También incluye pasos para proporcionar una base para la incorporación de servicios como Exchange Online, SharePoint Online y Microsoft Teams, incluida una explicación sobre seguridad, conectividad de red y cumplimiento.

    La incorporación de uno o más servicios elegibles puede empezar al finalizar la incorporación principal.

    Los especialistas de FastTrack proporcionan primero instrucciones remotas para las funcionalidades principales (comunes para todos los servicios en línea de Microsoft).

    Los puertos más comunes asociados con el protocolo FastTrack:

    Puerto 1214 (TCP/UDP): el puerto predeterminado utilizado por el protocolo FastTrack para la comunicación entre los clientes de Kazaa y otros nodos de la red FastTrack. Se utilizaba para buscar y compartir archivos entre usuarios.

    Puerto 6881 - 6889 (TCP/UDP): se utilizaban comúnmente para la transferencia de datos entre pares en la red FastTrack. Estos puertos podían configurarse en los clientes para facilitar la descarga y carga de archivos.

    Puertos del protocolo IRC.

    El protocolo IRC (Internet Relay Chat), es un sistema de chat en línea que se utiliza para la comunicación en tiempo real entre usuarios de todo el mundo.

    Es importante mencionar que este puerto no es el único utilizado por IRC, ya que algunos servidores pueden configurarse para utilizar otros puertos.

    IRC fue creado en 1988 por Jarkko Oikarinen, un estudiante finlandés de informática. Oikarinen desarrolló IRC como una forma de permitir la comunicación en tiempo real entre usuarios de diferentes redes de computadoras. Desde entonces, IRC ha evolucionado y se ha convertido en una de las formas más populares de comunicación en línea.

    Utiliza el siguiente puerto:

    Puerto 6667. (TCP) Para establecer conexiones con los servidores de IRC.

    IRC es un protocolo de chat en línea que utiliza el puerto 6667 para establecer conexiones con los servidores de IRC. Este sistema de conversación en línea utiliza canales para permitir la comunicación entre usuarios de todo el mundo. Los servidores de chat alojan estos canales y ofrecen la posibilidad de enviar mensajes privados a otros usuarios. Los chats privados son una forma de comunicación en línea entre dos usuarios de IRC.

    Puertos del protocolo XMPP.

    Está basada en servidores, pero descentralizada; por diseño, no hay ningún servidor central, como sucede con servicios como AOL Instant Messenger o MSN Messenger. Sobre este punto, surge cierta confusión, puesto que existe un servidor XMPP público en "Jabber.org", al que están suscritos un gran número de usuarios,

    Cada usuario en la red XMPP tiene un único identificador (Jabber ID, normalmente abreviado como JID). Para evitar la necesidad de un servidor central con una lista exhaustiva de identificadores, el Jabber ID está estructurado como una dirección de correo electrónico, con un nombre de usuario y una dirección DNS para el servidor en el que reside el usuario, separado por un signo @. Un identificador Jabber sería algo como nombredeusuario@dominio.com.

    Utiliza los puertos:

    Puerto 5222. (TCP): se utiliza para establecer conexiones de cliente a servidor y se emplea para enviar y recibir mensajes instantáneos, así como para administrar la presencia en línea y otras características relacionadas con la mensajería en tiempo real.

    Puerto 5269. (TCP): se utiliza para la comunicación entre servidores XMPP cuando se necesita el enrutamiento de mensajes entre dominios de servidor XMPP diferentes. Este puerto es importante para la interoperabilidad y el envío de mensajes entre servidores XMPP en diferentes dominios de Internet.

    Puertos del protocolo MTProto.

    Mobile Transport Protocol (Protocolo de transporte móvil) es el nombre del protocolo de datos de Telegram con que gestiona, cifra y transporta los mensajes, contactos y metadatos del servicio de mensajería. Está desarrollado bajo un estándar abierto, a base de API Java, desarrollado por el matemático Nikolái Dúrov y financiado por Pável Dúrov.

    MTProto está enfocado en la multisesión multiplataforma y el transporte de archivos sin importar su formato o capacidad. El tráfico tiene dos tipos de cifrados, ambos con AES de base, siendo los chat secretos para el envío de mensajes de forma restrictiva contra la filtración de información sensible.

  • MTProto v1:
    • La primera versión se estrenó en 2014. Los mensajes en MTProto fueron cifrados con el algoritmo SHA-1. Tras un reporte en enero de 2015, en donde el investigador Juliano Rizzo reveló un error en el funcionamiento de SHA-1 que originó una vulnerabilidad al interceptar los mensajes.Más adelante el medio De Welt en 2016 señaló un posible reemplazo SHA-2.

  • MTProto v2:
    • En 2017, se lanza la segunda versión. El cifrado se reemplazó a SHA-256 con mayor cantidad de bytes de carga útil. También está protegido de ataques IND-CCA.

    Utiliza el siguiente puerto:

    Puerto 443. (TCP): es el puerto estándar utilizado para el tráfico HTTPS seguro en la web, y Telegram utiliza este puerto para disfrazar su tráfico como tráfico web seguro, lo que dificulta que los proveedores de servicios de Internet bloqueen o restrinjan el acceso a Telegram.
